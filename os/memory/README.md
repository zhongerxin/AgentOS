# Memory

Memory stores durable project context that should survive individual chats and implementation sessions.

## Subfolders

- `chronicle/`: dated timeline entries.
- `decisions/`: decisions with rationale and consequences.
- `observations/`: facts noticed during work.
- `summaries/`: periodic summaries.

## Memory Rules

1. Store conclusions, not raw noise.
2. Prefer dated entries.
3. Link to source files when possible.
4. Keep decisions separate from observations.
5. When a memory becomes project truth, also update `wiki/`.
